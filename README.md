## 这个 Skill 能做什么

传统流程中，出题和建站是两个割裂的工具：你先手动出题，再找另一个工具建站。本 Skill 将两个阶段合并为一条流水线：

```
用户提供参考材料（课本、讲义、复习提纲等）
  → 阶段一：AI 自动出题，输出标准格式的 Markdown 题集
  → 阶段二：调用模板化 build.py，输出纯前端刷题网站（index.html）
```

最终产物是一个**单页 HTML 文件**，双击即可使用，不需要服务器、数据库、注册账号。

## 如何使用

### 一次性说完（推荐）

向 AI 输入类似以下指令，一次性说清全部需求：

> 「帮我把桌面上《心理学》复习提纲出成题。范围前三章，100 道单选加 50 道判断，解析详尽，简体字体，知识回溯到节一级。再做成刷题网站，六种模式全开，标题叫《心理学》期末刷题，风格你看着配。输出到桌面。」

AI 会直接走完全流程，无需追问。

### 分批交代

如果你只想先出题，日后再说建站的事：

> 「帮我把这份提纲出成题。」  
> → AI 追问材料路径、题型、题量等  
> → 出题完成  

日后想建站时：

> 「再帮我生成一个刷题网站。」  
> → AI 追问刷题模式、风格、标题  
> → 调用模板 build.py → 生成 index.html

## 出题规范

### 出题流程

1. **材料摸底** — AI 检查材料是否已转为可读格式。若为 PDF/扫描件，可先调用 OCR 工具处理
2. **需求收集** — 范围、题型、题量、解析风格、字体偏好等
3. **确认计划** — AI 给出题量分配方案，用户确认后执行
4. **逐批出题** — 按知识点分组出题，每批完成后用户可复核
5. **输出保存** — 按章节保存为独立的 `.md` 文件

### 干扰项质量标准

- 干扰项必须基于真实的、常见的错误理解
- 错误选项不能过于突兀，必须在形式上与正确选项一致
- 不定项选择题尽量不设置四项全对

## 建站规范

### 技术架构

- **纯前端**：单页 HTML，零后端依赖
- **原生 JS**：无框架依赖
- **Web Audio API**：点击、答对、答错音效
- **localStorage**：答题进度、错题集、生涯模式自动保存
- **Python 构建**：build.py 仅使用标准库，无第三方依赖

### 刷题模式

网站内置五种刷题模式：

| 模式 | 说明 |
|:-----|:------|
| **📖 专题模式** | 按章节顺序刷题，退出自动保存进度 |
| **🎲 随机模式** | 从总题库随机抽题，永不重复 |
| **🎯 生涯模式** | 答对永久移除，答错进错题池，全部通关为止 |
| **📝 错题专项** | 只显示之前做错的题，全部答对后清空 |
| **🏷️ 知识点专项** | 按「知识回溯」标签选择范围后刷题 |

### 交互特性

- 键盘快捷键：`Enter` 提交、`← →` 切换题目、`1-4` 选选项
- 连续答对 3 题以上显示 🔥 连对标记
- 所有按钮悬停放大、点按回弹
- 答对上升双音 / 答错低沉蜂鸣音效
- 响应式适配：手机端也可使用

### 视觉风格

三种预设风格，通过 `--style` 参数切换：

| 风格 | --style 值 | 特点 |
|:-----|:----------|:------|
| 古朴宣纸风 | `宣纸`（默认） | 暖色调、宋体、纸纹渐变 |
| 瑞士国际主义 | `瑞士` | 纯白底色、无衬线、蓝色强调色 |
| 极简卡片风 | `卡片` | 圆角卡片、柔和阴影、多色块 |

## 支持的三种题型

### 单选题

```markdown
### 第 1 题（单选题）

心理学是一门研究什么的科学？（ ）

A. 人的意识活动
B. 人的外部行为表现
C. **人的心理现象及其行为规律**
D. 人的思维过程

**答案**：C

**解析**：心理学是研究人的心理现象及其行为规律的科学。

**知识回溯**：第一章·心理学概述
```

### 不定项选择题

答案中用 ` · `（空格·空格）分隔多个字母，全选时标注「全選」：

```markdown
### 第 2 题（不定项选择题）

下列对「言」和「语」的辨析，正确的有（ ）

A. 「言」是自动跟人说话
B. 「语」是回答别人的问话或谈论事情
C. 「言」能带双宾语
D. 「语」可以带指谈话对方的宾语

**答案**：A · B · D

**解析**：……

**知识回溯**：常用词（一）「言」「语」
```

### 判断题

```markdown
### 第 3 题（判断题）

心理学的本质特征是有完整的故事情节。（ ）

**答案**：错误

**解析**：心理学的本质特征是……

**知识回溯**：第一章·心理学概述
```

## 更新题库

### 修改已有题目

直接编辑桌面上的 `.md` 文件（改答案、改错字、增删题目），然后告诉 AI：

> 「帮我重新生成一下刷题网站」

AI 会自动重新运行 build.py，生成新的 index.html。你也可以手动双击 `build.bat` 来完成同样的事。

### 追加新题目

有新章节或新材料时，告诉 AI：

> 「继续我上次的出题项目，《心理学》题库，追加第 11 章的内容」

AI 会读取之前保存在长期记忆中的项目状态（FACT.md），恢复用户偏好和进度，用新材料出题后追加到已有文件末尾，然后自动重建网站。

## 自定义与维护

### 轻度定制（改配色、改字体）

不需要修改 build.py。在调用命令中加入 `--custom-css` 参数即可覆盖任意 CSS 变量：

```bash
python build.py \
  --input "题库路径" \
  --title "标题" \
  --output "输出目录" \
  --custom-css ":root { --accent: #e74c3c; --bg-page: #fdf6ec; }"
```

### 深度定制（改布局、加功能）

复制 skill 目录下的 `build.py` 到项目目录中，直接修改：

- 改 CSS 变量 → 自定义配色方案
- 改 HTML_TEMPLATE → 调整页面布局
- 改 JS 逻辑 → 增加或修改交互功能

模板 build.py 中 90% 的代码是稳定的解析和交互逻辑，不需要动。只改需要改的部分即可。

### 版本历史

- **v1（指令型）**：AI 每次从零写 build.py，随机性 bug 反复出现
- **v2（模板型）**：build.py 模板化 + CLI 参数，一次编写永久复用；所有 bug 修复固化在模板中
