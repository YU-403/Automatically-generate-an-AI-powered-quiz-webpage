# Quiz Complete · 出题建站一体

> **本仓库是一个 AI Agent Skill，用于导入到兼容 Claude Agent 的客户端（如 CherryStudio 等）中使用。**
> AI 会按照本 Skill 定义的工作流，全自动完成从出题到建站的全部步骤。

---

## 这个 Skill 解决什么问题

期末复习时，你大概率遇到过这样的困境：课本翻了好几遍，但手头只有老师给的几道例题，想大量刷题却没有现成的题集；自己出题吧，对着教材一行一行找考点、手打题目和选项，费时费力；想做个刷题网页，又不会写前端代码。

这个 Skill 就是为了解决这个问题而诞生的——把「出题」和「建站」合并成一条流水线，让你从复习材料直接跳到刷题网站。

```
你的参考材料（课本 / 讲义 / 复习提纲）
  → 阶段一：AI 自动分析材料、按你的要求出题、输出标准格式的 Markdown 题集
  → 阶段二：调用模板化 build.py，自动生成纯前端刷题网站（index.html）
```

最终产物是一个**单页 HTML 文件**，双击即可使用，不需要服务器、不需要数据库、不需要注册账号、不需要写任何代码。

除了期末复习，老师、助教、培训师也可以用这个 Skill 快速为自己的学生生成题库和练习网站。如果你只需要其中某一个阶段（比如只出题不做网站，或者已有题库只想建站），也完全支持——分批交代即可。

### 环境要求

| 项目 | 必/可选 | 说明 |
|:-----|:-------:|:------|

| **浏览器** | ⚠️ 可选 | 用于打开生成的刷题网站；手头有浏览器即可，无须额外安装 |

### 材料准备

| 材料 | 必/可选 | 说明 |
|:-----|:-------:|:------|
| **参考材料** | ✅ 必选 | 你希望出题的依据，可以是教材扫描件、课堂讲义、复习提纲、Markdown 笔记等 |
| **出题需求** | ✅ 必选 | 考察范围（哪些章节）、题型偏好、题量、字体偏好、解析风格等 |

## 使用方法

### 方式一：一次性说完（推荐）

如果你已经想好了出题和建站的全部需求，可以一次性告诉 AI。AI 会直接走完全流程，无需追问。

> **示例指令：**
> 「帮我把桌面上《现代汉语》讲义出成题，范围前四章，100 道单选 + 50 道判断，解析详尽，简体字体，知识回溯到节一级。再做成刷题网站，六种模式全开，标题叫『现代汉语期末刷题』，风格用宣纸风。输出到桌面。」

### 方式二：分批交代

如果你只想先出题，日后再说建站的事：

> **第一轮：**「帮我把这份提纲出成题。」
> → AI 追问材料路径、题型、题量等
> → 出题完成，保存为 Markdown 文件

过几天想建站了：

> **第二轮：**「再帮我基于之前那份题库生成一个刷题网站。」
> → AI 追问刷题模式、风格、标题
> → 调用 build.py → 生成 index.html

你也可以反过来——已有现成题库，只想建站：

> **直接说：**「我桌面上有一个『题库』文件夹，里面是 Markdown 格式的题目，帮我生成刷题网站。模式开专题和随机，卡片风格，标题叫『语言学概论』。」

## 出题规范

### 出题流程

AI 在执行阶段一（出题）时，会按照以下步骤操作：

1. **材料摸底** — 检查你的参考材料是否已转为可读格式。若为 PDF 或扫描件，先调用 OCR 工具处理（需你确认）
2. **需求收集** — 逐一收集考察范围、题型、题量、解析风格、字体偏好、知识回溯粒度等；如果你有样题样本，在此阶段提供给 AI 作为风格参考
3. **确认计划** — AI 给出题量分配方案（各章节各题型各多少题），你确认后执行
4. **逐批出题 + 复核** — 按知识点分组出题，每出一批暂停请你复核；可中途要求修改
5. **输出保存** — 按章节保存为独立的 `.md` 文件，格式严格按照标准

### 干扰项质量标准

- 干扰项必须基于真实的、常见的错误理解（如古今义混淆、概念偷换等）
- 错误选项不能过于突兀，必须在形式上与正确选项一致，在语境中看似合理
- 不定项选择题尽量不设置四项全对
- 每个选项都有独立考察价值，无凑数选项

### 支持的三种题型

#### 单选题

```markdown
### 第 1 题（单选题）

下列不属于语法单位的是（ ）

A. 语素
B. 词
C. **短语**
D. 句子

**答案**：C

**解析**：短语是由词组成的语法单位，但题干问的是"不属于"语法单位的选项，实际上语素、词、短语、句子都是语法单位。本题考察语法单位的层级划分……

**知识回溯**：第二节·语法单位
```

#### 不定项选择题

答案中用 ` · `（空格·空格）分隔多个字母：

```markdown
### 第 2 题（不定项选择题）

下列语言单位中，属于"造句单位"的有（ ）

A. 语素
B. 词
C. 短语
D. 句子

**答案**：B · C · D

**解析**：语素是最小的音义结合体，是构词单位；词、短语、句子都是造句单位……

**知识回溯**：第二节·语法单位
```

#### 判断题

```markdown
### 第 3 题（判断题）

"词"是最小的能够独立运用的语言单位。（ ）

**答案**：正确

**解析**：词是语言中独立运用的最小音义结合体，这是语言学中对"词"的定义。

**知识回溯**：第二节·词的定义
```

## 建站规范

### 技术架构

| 特性 | 说明 |
|:-----|:------|
| **前端** | 纯原生 HTML + CSS + JavaScript，零框架依赖 |
| **音效** | Web Audio API 生成（点击音、答对上升双音、答错低沉蜂鸣） |
| **数据持久化** | 浏览器 localStorage（进度、错题集、生涯模式自动存档） |
| **构建脚本** | Python 3 标准库（os / re / json / hashlib / argparse），零第三方依赖 |
| **部署** | 单页 HTML，双击可用，也可部署到 GitHub Pages 等静态托管 |

### 交互特性

- 键盘快捷键：`Enter` 提交、`← →` 切换题目、`1-4` 选选项
- 连续答对 3 题以上显示 🔥 连对标记
- 所有按钮悬停放大（scale: 1.04）、点按回弹
- 提交答案后自动滚动到裁决区
- 答对 / 答错均展示确认条，配合弹跳动画
- 完成时撒纸屑庆祝动画 + 统计面板

### 六种刷题模式

默认启用前五种，考试模拟为可选（通过 `--modes` 参数控制）：

| 模式 | 标识 | 默认 | 说明 |
|:-----|:----|:----:|:-----|
| 📖 **专题模式** | `topic` | ✅ | 按板块分层选择，顺序刷题，退出自动保存进度 |
| 🎲 **随机模式** | `random` | ✅ | 从总题库随机抽题，永不重复，不保留进度 |
| 🎯 **生涯模式** | `career` | ✅ | 答对永久移除，答错进错题池，全部通关自动存档 |
| 📝 **错题专项** | `wrong` | ✅ | 只显示之前做错的题，全部答对后自动清空 |
| 🏷️ **知识点专项** | `tag` | ✅ | 按「知识回溯」标签选择知识点范围后刷题 |
| 📋 **考试模拟** | `exam` | — | 每次开始前弹窗输入自定义题数，完成后显示分数与统计 |

你可以在建站时指定只启用部分模式，未被启用的模式不会在网站中出现。

### 三种视觉风格

通过 `--style` 参数切换：

| 风格 | 参数值 | 特点 |
|:-----|:-------|:-----|
| 古朴宣纸风 | `宣纸`（默认） | 暖色调、宋体/黑体、纸纹径向渐变，适合文史类题库 |
| 瑞士国际主义 | `瑞士` | 纯白底色、无衬线字体、蓝色强调色，适合理工类题库 |
| 极简卡片风 | `卡片` | 圆角卡片、柔和阴影、多色块区分，通用风格 |

### CLI 参数说明

建站时 AI 会自动调用模板 build.py，你也可以手动执行：

```bash
python build.py \
  --input "题库路径" \
  --title "网站标题" \
  --output "输出目录"
```

全部参数：

| 参数 | 必填 | 说明 |
|:-----|:----|:------|
| `--input` | ✅ | 题库 Markdown 文件夹路径（含 .md 文件的目录） |
| `--title` | ✅ | 网站标题 |
| `--output` | ✅ | 输出目录（生成的 index.html 保存到此） |
| `--modes` | | 启用的模式，逗号分隔（默认: `topic,random,career,wrong,tag`） |
| `--style` | | 视觉风格（默认: `宣纸`，可选: `宣纸` / `瑞士` / `卡片`） |
| `--subtitle` | | 副标题，显示在首页标题下方 |
| `--katex` | | 启用 KaTeX 公式渲染（题库中含数学公式时使用） |
| `--custom-css` | | 自定义 CSS 变量覆盖（轻度定制配色，无需修改模板） |
| `--prefix` | | localStorage 存储前缀（通常自动生成，无需手动指定） |

### 版本历史

| 版本 | 特点 |
|:-----|:------|
| **v1（指令型）** | AI 每次从零编写 build.py，随机性 bug 反复出现 |
| **v2（模板型）** | build.py 模板化 + CLI 参数驱动，一次编写永久复用；所有 bug 修复固化在模板中 |
| **v2.1（稳定 ID）** | 基于题干哈希的稳定 ID 机制，增删题目不影响已有进度；修复动画冲突 |
| **v2.2（体验优化）** | 题型标签彩色化（蓝/橙/绿 Badge）、考试模式自定义题数、递归搜索子目录 .md 文件、板块按中文语义排序 |

### 自定义与维护

**轻度定制（改配色、改字体）：**

不需要修改 build.py。在调用命令中加入 `--custom-css` 参数即可覆盖任意 CSS 变量：

```bash
python build.py \
  --input "题库路径" \
  --title "标题" \
  --output "输出目录" \
  --custom-css ":root { --accent: #e74c3c; --bg-page: #fdf6ec; }"
```

**深度定制（改布局、加功能）：**

复制 skill 目录下的 `build.py` 到项目目录中，直接在副本上修改：

- 改 CSS 变量 → 自定义配色方案
- 改 `HTML_TEMPLATE` → 调整页面布局
- 改 JS 逻辑 → 增加或修改交互功能

模板中 90% 的代码是稳定的解析和交互逻辑，不需要动。只改需要改的部分即可。

## 更新题库

### 修改已有题目

直接编辑桌面上的 `.md` 文件（改答案、改错字、增删题目），然后告诉 AI：

> 「帮我重新生成一下刷题网站」

AI 会自动重新运行 build.py，生成新的 index.html。你也可以手动双击 `build.bat` 来完成同样的事。

### 追加新题目

有新章节或新材料时，告诉 AI：

> 「继续我上次的出题项目，追加第 11 章的内容」

AI 会读取之前保存在长期记忆中的项目状态，恢复你的偏好和进度，用新材料出题后追加到已有文件末尾，然后自动重建网站。

## 技术说明

| 项目 | 说明 |
|:-----|:------|
| 构建脚本 | Python 3（标准库，零第三方依赖） |
| 网站前端 | 原生 HTML + CSS + JavaScript（零框架依赖） |
| 题库格式 | Markdown（题型标注清晰，便于人工审阅和编辑） |
| 题目 ID | 基于题干内容哈希生成（`q_` + md5 前 8 位），增删调序不影响已有进度 |
| 版本检测 | build.py 自动注入版本哈希，题库更新时页面顶部提示"题库已更新" |
| 数据持久化 | 浏览器 localStorage（按输出文件夹名隔离，多题库互不干扰） |
| 音效 | Web Audio API（零音频文件，纯代码合成） |
| 输出产物 | 单页 HTML（约 300–600 KB，双击可用，可部署到 GitHub Pages） |
