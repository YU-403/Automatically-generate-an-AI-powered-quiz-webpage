# Quiz Complete · 出题建站一体

**从参考材料到刷题网站，一站式生成。**

本仓库是一个面向大学生自学与复习场景的 **AI Agent Skill 蓝图**。它定义了一套完整的工作流程：用户只需提供参考材料（课本、讲义、复习提纲等），AI 就能自动完成出题 → 建站全流程，最终产出一个可直接运行的纯前端刷题网站。

适用于：师范生备考教资、考研复习、期末冲刺、自考刷题等场景。

## 设计理念

### 为什么要合并"出题"和"建站"？

传统刷题工具链的问题是割裂的——出题是出题，建站是建站，用户需要在两个工具之间来回搬运数据。实际上，这两个环节共享同一份题库格式、同一套质量规范，天然应该是一条流水线。

本 Skill 将两个阶段合并为一条工作流：

```
材料 → 出题（Markdown 题集）→ 建站（刷题网站 HTML）
       ↑ 共用格式规范 ↓
   题目答案解析知识回溯 ← build.py 解析构建
```

### 设计原则

- **一次性说完，也可以分批说**：用户可在一次输入中交代全部需求，模型自动执行；信息不足时再逐项追问，不增加沟通冗余。
- **质量优先于速度**：出题阶段采用逐批生成 + 用户复核机制，优先保证单题质量。
- **纯前端，零部署成本**：最终产物是一个单页 HTML 文件，不需要服务器、数据库、注册账号，双击即可使用。
- **持久化本地进度**：答题进度、错题集、生涯模式存档均存储于浏览器 localStorage，关闭页面后不丢失。

## 使用方式

本仓库是一个 **AI Agent Skill 的指令蓝图**，不是可以直接运行的程序。你需要将其导入到兼容 Claude Agent 的客户端中（如 CherryStudio 等），让 AI 按照蓝图中的工作流为你执行。

### 基本用法

向 AI 输入类似以下指令：

> **一次性说完（推荐）：**
> ```
> 帮我把 D:\复习\心理学提纲.md 出成题。范围是前三章，100 道单选 + 50 道判断，
> 解析详尽，简体字体，知识回溯到节一级。然后直接做成刷题网站，六种模式全开，
> 标题叫"《心理学》期末刷题"，风格你看着搭配。输出到桌面。
> ```
> → AI 无需追问，直接走完全流程。

> **分批交代：**
> ```
> 我：帮我把这份提纲出成题。
> AI：材料路径是什么？……
> 我：路径是 XXX，出单选和判断，共 200 道。
> AI：题型比例？解析风格？……
> （出完题后）
> 我：再帮我生成个刷题网站。
> AI：刷题模式？视觉风格？……
> ```

### 输出产物

运行完成后，会在指定目录生成以下文件：

```
心理学题库/
├── index.html          # 刷题网站（双击用浏览器打开）
├── build.py            # 构建脚本（修改题库后重新生成）
├── build.bat           # 一键构建（Windows）
├── README.md           # 使用说明
├── 第一章-心理学概述.md # 题库源文件（Markdown）
├── 第三章-认知心理.md   # 题库源文件
└── ...                 # 其他章节
```

## 出题规范

### 题目单元格式

每道题由四个部分组成：

```markdown
### 第 1 题（单选题）

题干文字（ ）

A. 选项一
B. 选项二
C. **正确选项**
D. 选项三

**答案**：C

**解析**：正确答案的依据和干扰项的错因说明。

**知识回溯**：第一章·心理学的概念
```

判断题格式（无选项，直接标记正误）：

```markdown
### 第 2 题（判断题）

题干文字（ ）

**答案**：错误

**解析**：……

**知识回溯**：第三章·注意的定义
```

### 干扰项设计原则

- 干扰项必须基于真实的、常见的错误理解（如古今义混淆、概念偷换、同词多义的错误义项等）
- 不定项选择题尽量不设置四项全对的选项
- 错误选项不能过于突兀，必须与正确选项在形式上一致、在语境中看似合理
- 需要考虑学科本身的性质来设置干扰项

### 输出结构

按章节分组，每个章节独立一个 `.md` 文件，便于用户集中复习和后续修改。

```
桌面/心理学题库/
├── 第一章-心理学概述.md
├── 第三章-认知心理.md
├── 第四章-学习心理.md
└── ...
```

## 建站规范

### 技术架构

- **纯前端**：单页 HTML，零后端依赖
- **JavaScript 原生**：无框架依赖（Vue/React 等），无第三方包
- **CSS 原生**：无预处理器
- **Web Audio API**：点击、答对、答错音效
- **localStorage**：答题进度、错题集、生涯模式存档
- **Python 构建**：`build.py` 使用标准库（os, re, json），无第三方依赖

### 刷题模式

网站内置六种刷题模式，覆盖不同复习场景：

| 模式 | 说明 | 默认 |
|:-----|:------|:----:|
| **📖 专题模式** | 按章节分层选择，顺序答题，退出自动保存进度 | ✅ 启用 |
| **🔀 随机模式** | 从总题库随机抽题，永不重复，不保留进度 | ✅ 启用 |
| **🏆 生涯模式** | 答对永久移除，答错进入错题池，全部通关为止 | ✅ 启用 |
| **📝 错题专项** | 只显示之前做错的题，全部答对后清空 | 可选 |
| **🏷️ 知识点专项** | 按「知识回溯」标签选择范围后刷题 | 可选 |
| **📋 考试模拟** | 随机抽取 30 题，全部答完后显示分数 | 可选 |

### 交互特性

- 提交答案后自动滚动到结果区域
- 答案/解析/知识回溯三个区块有独立的视觉风格，一目了然
- 连续答对 3 题以上显示连对标记 🔥
- 所有按钮悬停放大、点按回弹
- 答对上升双音 / 答错低沉蜂鸣

### 响应式适配

支持手机端访问，但核心体验针对桌面端设计。

### 视觉风格

预置三种风格，也支持用户自定义：

| 风格 | 特点 |
|:-----|:------|
| **古朴宣纸风** | 暖色调、宋体/黑体、纸纹渐变 |
| **瑞士国际主义** | 纯白底色、无衬线字体、单一强调色 |
| **极简卡片风** | 圆角卡片、柔和阴影、多色块区分 |

## 工程经验

本项目在多次真实协作中积累了大量工程经验，已汇总为内置的"踩坑记录"：

| 问题 | 原因 | 预防 |
|:-----|:-----|:------|
| 题干解析失败 | 正则只匹配半角 `)`，中文用全角 `）` | 兼容 `[^)）]*[)）]` |
| 导航按钮提前可见 | 未检查提交状态 | 提交后显示导航 |
| 答案/解析/回溯视觉雷同 | 共用同一套样式 | 独立 CSS 类区分 |
| 题目编号重复 | 同一文件两个"第 N 题" | 解析时检测重复 |
| 简繁「知识回溯」不兼容 | 正则为单一一套写法 | `(?:知识\|知識)` 分组匹配 |
| 正确选项两处不一致 | 加粗标记与答案字段矛盾 | 解析后交叉校验 |
| 多题库进度覆盖 | localStorage key 硬编码 | 文件夹名作前缀隔离 |
| 提交后看不到结果区 | 题目长时区域在屏幕下方 | 自动 scrollIntoView |
| HTML 字符渲染异常 | `<` `>` `&` 被当作标签解析 | esc() 转义 |
